Turbine is a simple web based tool which is capable of generating complex and life-like 3d models and animation.

It was written at ITP (NYU's interactive telecommunications program) for Daniel Shiffman's class, "The Nature of Code"

Web based, real-time, obj exporter tuned for 3d printing, animating

Turbine is built with threejs, an open framework being written/curated by Mr. Doob.
It also makes use of dat.gui for interface elements.
The instructions a computer needs for making a tree are quite simple:

1.) Build a branch section, in this case it's a simple cylinder parented under null objects to create a rig that's capable of complex motion and offsets.

2.) Create a recursive function that generates and parents a long row of those limbs under each other. The results from having just one trunk are remarkable in their variety.

3.) As the trunk is being created, for every nth item spawn a new limb with an offset orientation.


The major advantage to building a tree with joint sections is the fine control over bending, allowing for some extremely complex curves. Some very typical fractal like geometry can be constructed, as can shapes that are more unusual.

What I find most remarkable about this setup is its simplicity. There is absolutely nothing original here (in fact it has a lot in common with the more complete structure synth), but a few modifications in the basic structure of a typical L-system and (I think) a sensibility born of animation and design has allowed me to produce idiosyncratic geometry. The controls allow you to think about building structures with these basic rules:

1.) Set the number of layers deep branches can be grown.
2.) Set the number of joints on each branch.
3.) Set the frequency of branching (for every nth joint, make a branch).
4.) Set the # of radial repetitions. You could, for instance, make a bowl if you set the repeats high enough.
5.) Set the scale factor on successive joints, scaling up or down over the length of a branch.
6.) Bend and rotate each branch and each joint within each branch.
7.) Add fruit at the end of selected layers of leaves.

Bonus:
8.) The user can write their own custom animation functions and pass them directly through the params.
9.) Geometry can be replaced per branch.

Turbine allows for some extremely complicated shapes to arise from extremely simple inputs, the same as any branching/fractal system. Turbine is unique in its animated approach and the fact that it's web based and extremely simple.

For the Future:

1.) I'd like to add the ability for all the joints to bend in the same direction, this will be as simple as rotating around a specific axis.
2.) A database will be implemented so users can save their projects.
3.) A connection directly to Shapeways will be added.
4.) Users will be able to upload their own geometry to be instanced along the joints. For instance, vertebrae for the branches and skulls for the fruit, or for something more fun, gummy bears for
branches and lollipops for fruit.
5.) The interface will be simplified considerably, I will likely hide the params section (or only keep it as an advanced tab) and replace all of those elements with sliders.

Screen Left Pane
Presets: Choose from the dropdown, then click 'applyPreset'
ApplyPreset: Click here after you choose a preset
Rebuild: Uses the code in the params window to rebuild the tree
SliderToParams: Takes the settings on the sliders (to the right) and copies them to this field

Parameters: For advanced users, you can adjust all the settings here and then click 'rebuild'
  • ss: how much each link is scaled
  • sc: initial ss of the root on construction
  • scale: x,y,z scale of the root on construction
  • jointScale: x,y,z scale of each leaf layer, -1 sets it to defaults, these branches will not get bigger in x than the parent
  • num: number of joints in a branch
  • divs: how often new branches sprout from a parent branch
  • rads: when the branches branch, how many sprout
  • x1,y1,z1: the bend of the branches in an array
  • x2,y2,z2: the rotation of the root of the branches
  • x3,y3,z3: animation values in an array
  • x4,y4,z4: offset based on position along parent
  • off: sin offset along parent branch
SaveOBJ: export .obj file
HelperGeo: Places tubes inside the geo to demonstrate a safe minimum thickness for 3d printing

Sliders

Constructor:
TrunkDivisions: number of joints in a branch
TrunkScale: each successive joint is scaled by this amount
Leaves: depth of the branching
Rebuild: same as the rebuild in the left pane, but it copies over slider values first
fruitSize: scale of fruit

Motion:
speed/speedmultiply: two sliders are present for fine tuning
wavelength: of the sine wave used to rotate the joints
animate x,y,z: this is a multiplier for the animate-x,y,z sliders on each branch

Level: (refers to branch level)
Bend X,Y,Z: rotates each joint
Rotate X,Y,Z: rotates only the root of each jonit
Animate X,Y,Z: amount of animation, multiplies with sliders in the 'motion' tab
Offset X,Y,Z: similar to animate but it's static and uses the position along the parent to control the offset of the sine wave
OffsetMultiply: similar to the 'wavelength' attribute in the motion section
Scale: 'ss' in the params, the successive scale of each joint
